In React erfolgt die **Ereignisbehandlung** (Event Handling) √ºber Props, die Event-Handler-Funktionen enthalten. Diese Props haben √§hnliche Namen wie die nativen DOM-Ereignisse, jedoch in CamelCase-Notation (z.‚ÄØB. `onClick` statt `onclick`).

**Wie funktioniert das in unserem Beispiel?**

- **`Cell`-Komponente:**

  - Die `Cell`-Komponente erh√§lt eine `onClick`-Prop, die eine Funktion ist.
  - Beim Klick auf die Zelle wird die `onClick`-Funktion aufgerufen:

  - Die `handleCellClick`-Funktion verarbeitet den Klick auf eine Zelle.
  - Sie pr√ºft, ob die Zelle leer ist und ob es bereits einen Gewinner gibt.

**Warum verwenden wir `onClick={() => handleCellClick(index)}`?**

- Wir m√∂chten der `Cell`-Komponente mitteilen, welche Zelle geklickt wurde.
- Durch die Verwendung einer anonymen Funktion `() => handleCellClick(index)` erzeugen wir eine neue Funktion, die bei Klick ausgef√ºhrt wird und den Index der geklickten Zelle an `handleCellClick` √ºbergibt.
